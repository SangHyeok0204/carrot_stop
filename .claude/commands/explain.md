# explain

프로젝트 온보딩용 설명 커맨드. 신입 개발자가 프로젝트에 빠르게 적응할 수 있도록 설명한다.

## 수행 작업

현재 프로젝트의 구조와 흐름을 실무 중심으로 설명한다.

1. **프로젝트 전체 파악**
   - 디렉토리 구조 분석
   - 주요 설정 파일 확인 (package.json, tsconfig, 환경설정 등)
   - 핵심 진입점과 라우팅 구조 파악

2. **데이터 흐름 추적**
   - 사용자 액션 → API 호출 → 데이터 처리 → UI 반영 흐름
   - 상태 관리 방식과 Context/Store 구조
   - 외부 서비스 연동 포인트

3. **실제 예시 기반 설명**
   - 구체적인 기능 하나를 선택해 처음부터 끝까지 흐름 설명
   - 코드 경로와 함께 설명

## 출력 구조

```markdown
## 1. 프로젝트 한 줄 요약
[프로젝트가 무엇을 하는지 한 문장으로]

## 2. 전체 구조
[디렉토리별 역할과 핵심 파일 설명]

## 3. 주요 흐름 예시
[구체적인 시나리오 기반 step-by-step 설명]
예: "사용자가 캠페인을 생성하면..."
1. 페이지 진입: /advertiser/campaigns/new
2. 폼 제출 → API 호출: POST /api/campaigns
3. 데이터 저장: Firestore campaigns 컬렉션
4. 상태 업데이트 → UI 반영

## 4. 신입이 헷갈릴 포인트
[자주 실수하거나 이해하기 어려운 부분 정리]
- 왜 이렇게 설계했는지
- 주의해야 할 패턴
- 흔한 실수와 해결법
```

## 주의사항

- 교과서적/이론적 설명 지양
- 실제 코드 경로와 파일명을 구체적으로 언급
- "왜 이렇게 했는지" 설계 의도 중심 설명
