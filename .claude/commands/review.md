# review

실서비스 기준 코드 리뷰 커맨드. 프로덕션 환경에서 발생할 수 있는 실질적 문제에 집중한다.

## 수행 작업

다음 관점에서 코드를 검토한다:

1. **확장성 (Scalability)**
   - 데이터가 많아졌을 때 성능 저하 가능성
   - N+1 쿼리, 불필요한 재렌더링, 메모리 누수 패턴

2. **유지보수성 (Maintainability)**
   - 코드 수정 시 사이드 이펙트 발생 가능성
   - 암묵적 의존성, 하드코딩된 값
   - 테스트하기 어려운 구조

3. **안정성 (Reliability)**
   - 에러 핸들링 누락
   - 엣지 케이스 미처리 (null, undefined, 빈 배열 등)
   - 경쟁 조건 (race condition) 가능성

4. **보안 (Security)**
   - 인증/인가 누락
   - 입력값 검증 부재
   - 민감 정보 노출 가능성

## 리뷰하지 않는 것

- 코드 스타일, 포맷팅, 네이밍 컨벤션
- 개인 취향 수준의 구조 차이
- 동작에 영향 없는 사소한 개선점

## 출력 형식

각 이슈마다 다음 형식으로 작성:

```markdown
### [심각도] 이슈 제목
**위치**: 파일명:라인번호

**문제점**
[현재 코드에서 발생할 수 있는 구체적인 문제 상황]

**이유**
[왜 이것이 실서비스에서 문제가 되는지]

**개선 방향**
[어떻게 수정하면 좋은지 구체적인 방향 제시]
```

심각도 기준:
- `CRITICAL`: 즉시 수정 필요 (장애, 보안, 데이터 손실)
- `HIGH`: 빠른 수정 권장 (성능 저하, 버그 가능성)
- `MEDIUM`: 개선 권장 (유지보수성, 확장성)
